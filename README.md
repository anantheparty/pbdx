# 拼豆图纸生成器 · Bead Pattern Studio

把任意图片转成可施工的拼豆 / 熔珠 / Perler / Hama 图纸的纯前端工具。
所有运算都在浏览器里完成，不上传图片、不需要后端。

## 运行

无构建步骤，本地起一个静态服务器即可：

```bash
npm start         # 等价于 python3 -m http.server 5173
# 浏览器打开 http://localhost:5173
```

修改源码后浏览器需要手动刷新（这个仓库没有 hot-reload）。

`npm run check` 会用 `node --check` 跑一遍每个模块做语法体检。

部署：纯静态站点，`netlify.toml` / `vercel.json` 已经放好，直接把整个目录当根目录发布即可，GitHub Pages 同理。

## 功能一览

### 图片到图纸
- 内置多家色卡标准：Artkal M-Mini 221、MARD 221 / 264 兼容集 / 291 等；Artkal M-Mini 221 显示时会自动去掉开头的 `M` 前缀，看起来跟实物色号一致
- 支持导入自定义色卡（JSON 或 CSV，列 `code, hex` 或 `code, r, g, b`）
- 自动按 完整保留 / 裁切铺满 / 强制拉伸 做尺寸适配，可锁定原图比例
- 透明阈值 + "近白留空"边缘连通检测：默认只挖与画布边缘连通的近白背景，保留人物内部高光
- 色差算法可选 CIEDE2000（推荐）或快速 Lab76
- 抖动：不用 / 有序抖动 / 误差扩散
- 五个施工友好预设：高还原、平衡、省事大块、低色数、点绘像素
- 同色成块、孤岛清理、边缘保护等参数都有滑杆细调
- **孤豆合并色差上限**：孤立小色块只有在颜色和邻近差异 ≤ 阈值时才合并，眼睛、嘴巴这种 1～2 像素但对比强烈的细节会被保留下来
- 勾选"参数修改后自动重新生成"后，调滑杆会带防抖地实时出图

### 编辑与查看
- 画笔 / 橡皮 / 吸管 / 移动 四种工具，撤销重做
- macOS 触控板原生手势：双指捏合缩放、双指拖动平移
- 滚轮缩放灵敏度可调（每次 1.02～1.30 倍）
- 画布高度可拖拽调整，记忆到 localStorage
- 板线分粗细两档（默认每 10 格粗线 + 每 5 格细线）
- 格内显示色号、行列坐标
- "高亮当前色号"会把当前选中颜色之外的格子整体暗化，便于核对一种颜色的分布
- 高色差提示：原图与最终色号差异较大的格子会标出来

### 用豆统计与全局替换
- 实时列出每个色号的用量
- 排序方式：数量 ↑↓、按色相、按色号
- 可以选一个目标色把当前选中色号全局替换掉，支持撤销
- 一键导出用豆 CSV

### 导入导出
- 自有 `.pbdx` 格式（JSON + RLE），保留全部图纸数据可二次编辑
- PNG / SVG 图纸导出
- 总览 HTML（含色号表）
- 用豆 CSV

## .pbdx 格式

```json
{
  "type": "BeadPatternX",
  "version": 1,
  "width": 96,
  "height": 96,
  "palette": { "id": "...", "colors": [] },
  "cellsRle": [[-1, 12], [5, 3]],
  "params": {},
  "metrics": {}
}
```

格子数据用 RLE 压缩：`[色卡索引, 连续数量]`。`-1` 表示空格 / 不放豆。

## 自定义色卡格式

JSON：

```json
{
  "id": "my_palette",
  "label": "我的色卡",
  "colors": [
    { "code": "A1", "name": "A1", "hex": "#FAF4C8" },
    { "code": "A2", "rgb": [255, 255, 213] }
  ]
}
```

CSV：

```csv
code,name,hex
A1,A1,#FAF4C8
A2,A2,#FFFFD5
```

或：

```csv
code,name,r,g,b
A1,A1,250,244,200
```

## 施工参数小抄

- 想更像原图：提高最大用色数，降低同色成块强度，减少清理。
- 想更好摆豆：降低最大用色数，提高同色成块强度，提高孤岛最小保留面积，把孤豆合并色差上限调大一点。
- 不建议默认开启抖动，会制造很多散豆；它适合追求像素画颗粒质感，不适合省事施工。
- 白底图片调"近白留空阈值"；默认只清除与画布边缘连通的近白背景，尽量保留角色内部高光。

更多标准说明见 `docs/STANDARD.md`。
